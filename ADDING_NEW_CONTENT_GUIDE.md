## 1. إضافة أحداث جديدة

الأحداث التاريخية تُخزن في ملف `src/dataList.json` بصيغة JSON Array.

### بنية JSON للأحداث

```json
{
  "id": "unique-event-id",
  "category": "أحداث",
  "era": "العهد المكي",
  "title": "عنوان الحدث",
  "is_major_event": true,
  "date": {
    "hijri": -53,
    "hijri_relative": "عام الفيل",
    "gregorian": 571
  },
  "location": {
    "name": "اسم المكان",
    "coordinates": [21.4225, 39.8262]
  },
  "details": {
    "summary": "ملخص مختصر للحدث",
    "full_description": "وصف تفصيلي كامل للحدث",
    "course_of_events": [
      "الخطوة الأولى",
      "الخطوة الثانية",
      "الخطوة الثالثة"
    ],
    "companion_roles": [
      {
        "companion_id": "abu-bakr",
        "role": "دوره في الحدث"
      }
    ]
  },
  "entities": {
    "key_figures": ["شخصية 1", "شخصية 2"],
    "quran_refs": ["سورة الفيل: (أَلَمْ تَرَ...)"],
    "hadith_refs": ["صحيح مسلم: نص الحديث"],
    "sources": ["المصدر الأول", "المصدر الثاني"]
  }
}
```

### الحقول المطلوبة والاختيارية

#### حقول إلزامية:
- `category`: نوع الحدث ("أحداث" أو "battle")
- `era`: العصر ("العهد المكي" أو "العهد المدني" أو "عهد الخلفاء الراشدين")
- `title`: عنوان الحدث
- `date`: كائن يحتوي على التاريخ
  - `gregorian`: السنة الميلادية (رقم)
  - `hijri`: السنة الهجرية (رقم، سالب للأحداث قبل الهجرة)
- `location`: كائن يحتوي على:
  - `name`: اسم المكان
  - `coordinates`: مصفوفة [خط العرض، خط الطول]
- `details`: كائن يحتوي على:
  - `summary`: ملخص مختصر
  - `full_description`: وصف تفصيلي

#### حقول اختيارية:
- `id`: معرف فريد (يُفضل استخدامه للأحداث المهمة)
- `is_major_event`: منطقي (true/false) لتمييز الأحداث الكبرى
- `date.hijri_relative`: وصف نسبي للتاريخ (مثل "عام الفيل")
- `details.course_of_events`: مصفوفة بمراحل الحدث
- `details.companion_roles`: مصفوفة بأدوار الصحابة
- `entities`: كائن يحتوي على:
  - `key_figures`: شخصيات رئيسية
  - `quran_refs`: مراجع قرآنية
  - `hadith_refs`: مراجع حديثية
  - `sources`: المصادر التاريخية

### كيفية تحديد الإحداثيات الجغرافية

1. استخدم Google Maps أو OpenStreetMap
2. انقر بزر الماوس الأيمن على الموقع
3. انسخ الإحداثيات بصيغة [خط العرض، خط الطول]
4. تأكد من الترتيب: [Latitude, Longitude]

مثال:
- مكة المكرمة: `[21.4225, 39.8262]`
- المدينة المنورة: `[24.4672, 39.6024]`

### كيفية ربط الأحداث بالصحابة

استخدم حقل `companion_roles` داخل `details`:

```json
"companion_roles": [
  {
    "companion_id": "abu-bakr",
    "role": "رافق النبي ﷺ في الهجرة وكان معه في الغار"
  },
  {
    "companion_id": "ali",
    "role": "نام في فراش النبي ﷺ ليلة الهجرة"
  }
]
```

تأكد من استخدام `id` الصحيح للصحابي من ملفات `companionsList.ts`.

### كيفية إضافة مراجع قرآنية وحديثية

```json
"entities": {
  "quran_refs": [
    "سورة التوبة: (إِلَّا تَنصُرُوهُ فَقَدْ نَصَرَهُ اللَّهُ...)",
    "سورة الأنفال: (وَإِذْ يَعِدُكُمُ اللَّهُ إِحْدَى الطَّائِفَتَيْنِ...)"
  ],
  "hadith_refs": [
    "صحيح البخاري: كتاب المغازي، باب غزوة بدر",
    "صحيح مسلم: كتاب الجهاد والسير"
  ],
  "sources": [
    "سيرة ابن هشام",
    "الرحيق المختوم",
    "البداية والنهاية لابن كثير"
  ]
}
```

## 2. إضافة صحابة وشخصيات جديدة

الصحابة والشخصيات موزعة على خمسة ملفات:

1. `src/companionsList.ts` - الصحابة الأساسيون (الخلفاء الراشدون والعشرة المبشرون)
2. `src/companionsDataExtend.ts` - الملف الموسع الأول
3. `src/companionsDataExtend2.ts` - الملف الموسع الثاني
4. `src/companionsDataExtend3.ts` - الملف الموسع الثالث
5. `src/companionsDataExtend4.ts` - الملف الموسع الرابع

### بنية CompanionData

```typescript
{
  id: "unique-companion-id",
  name: "الاسم الكامل",
  title: "اللقب",
  role: "الدور الرئيسي",
  description: "وصف تفصيلي عن الصحابي",
  birth_death: "51 ق.هـ - 13 هـ",
  aliases: ["الاسم الأول", "الاسم الثاني", "اللقب"]
}
```

### الحقول المطلوبة

جميع الحقول التالية إلزامية:

- `id`: معرف فريد بالإنجليزية (استخدم حروف صغيرة وشرطات)
- `name`: الاسم الكامل بالعربية
- `title`: اللقب أو الكنية
- `role`: الدور الرئيسي أو المنصب
- `description`: وصف تفصيلي (فقرة أو أكثر)
- `birth_death`: تواريخ الميلاد والوفاة
- `aliases`: مصفوفة بالأسماء البديلة للبحث

### أهمية الـ aliases للبحث

حقل `aliases` مهم جداً لأنه يُستخدم في البحث. أضف جميع الأشكال الممكنة للاسم:

```typescript
aliases: [
  "أبو بكر الصديق",
  "أبو بكر",
  "أبي بكر",
  "الصديق",
  "عبد الله بن أبي قحافة"
]
```

### كيفية اختيار الملف المناسب للإضافة

1. **companionsList.ts**: للصحابة الأساسيين فقط (الخلفاء الراشدون، العشرة المبشرون)
2. **companionsDataExtend.ts**: للصحابة المشهورين (أمهات المؤمنين، كبار الصحابة)
3. **companionsDataExtend2.ts**: للصحابة المعروفين
4. **companionsDataExtend3.ts**: للصحابة الأقل شهرة
5. **companionsDataExtend4.ts**: لباقي الصحابة والشخصيات التاريخية

القاعدة: ابدأ من الملف الأخير (extend4) إلا إذا كان الصحابي مشهوراً جداً.

### مثال على إضافة صحابي جديد

في ملف `companionsDataExtend4.ts`:

```typescript
export const moreCompanions4: CompanionData[] = [
  // ... الصحابة الموجودون
  {
    id: "new-companion-id",
    name: "الاسم الكامل",
    title: "اللقب",
    role: "الدور",
    description: "وصف تفصيلي يشمل نسبه، إسلامه، مشاركته في الغزوات، مناقبه، ووفاته.",
    birth_death: "؟ - 40 هـ",
    aliases: ["الاسم", "اللقب", "الكنية"]
  }
];
```

## 3. إضافة مدن جديدة

المدن تُخزن في ملف `src/citiesList.ts`.

### بنية CityData

```typescript
{
  id: "unique-city-id",
  name: "اسم المدينة",
  coordinates: [latitude, longitude],
  description: "وصف المدينة وموقعها",
  significance: "أهميتها التاريخية والدينية"
}
```

### الحقول المطلوبة

جميع الحقول إلزامية:

- `id`: معرف فريد بالإنجليزية
- `name`: اسم المدينة بالعربية
- `coordinates`: مصفوفة [خط العرض، خط الطول]
- `description`: وصف عام للمدينة
- `significance`: الأهمية التاريخية والدينية

### مثال على إضافة مدينة جديدة

```typescript
export const citiesData: CityData[] = [
  // ... المدن الموجودة
  {
    id: "basra",
    name: "البصرة",
    coordinates: [30.5085, 47.7835],
    description: "مدينة عراقية أسسها عتبة بن غزوان في عهد عمر بن الخطاب.",
    significance: "مركز تجاري وعلمي مهم، ومنها انطلقت الفتوحات الإسلامية نحو فارس."
  }
];
```

## 4. نصائح مهمة

### كيفية تجنب التكرار

1. **ابحث أولاً**: قبل إضافة أي محتوى، ابحث في الملفات الموجودة
2. **استخدم البحث في الكود**: استخدم Ctrl+F أو Cmd+F للبحث عن الاسم أو الحدث
3. **تحقق من الـ id**: تأكد أن الـ id فريد ولم يُستخدم من قبل

### كيفية التحقق من عدم وجود البيانات مسبقاً

```bash
# للبحث عن حدث
grep -r "عنوان الحدث" src/dataList.json

# للبحث عن صحابي
grep -r "اسم الصحابي" src/companions*.ts

# للبحث عن مدينة
grep -r "اسم المدينة" src/citiesList.ts
```

### أهمية الدقة التاريخية

1. **تحقق من التواريخ**: استخدم مصادر موثوقة للتواريخ الهجرية والميلادية
2. **دقة الإحداثيات**: تأكد من صحة الموقع الجغرافي
3. **صحة المعلومات**: راجع المعلومات من مصادر متعددة
4. **الأمانة العلمية**: اذكر المصادر التي اعتمدت عليها

### مصادر موثوقة للمعلومات

1. **السيرة النبوية**:
   - سيرة ابن هشام
   - الرحيق المختوم (صفي الرحمن المباركفوري)
   - زاد المعاد (ابن القيم)

2. **التاريخ الإسلامي**:
   - البداية والنهاية (ابن كثير)
   - تاريخ الطبري
   - الكامل في التاريخ (ابن الأثير)

3. **تراجم الصحابة**:
   - الإصابة في تمييز الصحابة (ابن حجر)
   - أسد الغابة (ابن الأثير)
   - الاستيعاب (ابن عبد البر)

4. **المواقع الإلكترونية**:
   - موقع الدرر السنية
   - موقع إسلام ويب
   - موقع قصة الإسلام

### كيفية اختبار البيانات الجديدة

1. **تحقق من صحة JSON**:
```bash
# استخدم أداة للتحقق من صحة JSON
node -e "JSON.parse(require('fs').readFileSync('src/dataList.json'))"
```

2. **تحقق من صحة TypeScript**:
```bash
npm run build
```

3. **اختبر في المتصفح**:
   - شغل التطبيق: `npm run dev`
   - ابحث عن المحتوى الجديد
   - تحقق من ظهوره على الخريطة
   - تحقق من صحة التواريخ في الخط الزمني

## 6. الأخطاء الشائعة وكيفية تجنبها

### أخطاء في التنسيق

**خطأ**: نسيان الفاصلة بين العناصر
```json
{
  "title": "الحدث الأول"
  "date": { }  // خطأ: لا توجد فاصلة
}
```

**صحيح**:
```json
{
  "title": "الحدث الأول",
  "date": { }
}
```

**خطأ**: فاصلة زائدة في نهاية المصفوفة
```json
{
  "aliases": [
    "الاسم الأول",
    "الاسم الثاني",  // خطأ: فاصلة زائدة
  ]
}
```

**صحيح**:
```json
{
  "aliases": [
    "الاسم الأول",
    "الاسم الثاني"
  ]
}
```

### أخطاء في الإحداثيات

**خطأ**: عكس ترتيب خط العرض والطول
```json
"coordinates": [39.8262, 21.4225]  // خطأ: الطول أولاً
```

**صحيح**:
```json
"coordinates": [21.4225, 39.8262]  // صحيح: العرض أولاً
```

**خطأ**: استخدام إحداثيات غير دقيقة
```json
"coordinates": [21.0, 40.0]  // خطأ: تقريبي جداً
```

**صحيح**:
```json
"coordinates": [21.4225, 39.8262]  // صحيح: دقيق
```

### أخطاء في التواريخ

**خطأ**: استخدام نص بدلاً من رقم
```json
"date": {
  "hijri": "خمسة",  // خطأ: يجب أن يكون رقماً
  "gregorian": "632"  // خطأ: يجب أن يكون رقماً
}
```

**صحيح**:
```json
"date": {
  "hijri": 5,
  "gregorian": 632
}
```

**خطأ**: نسيان السالب للأحداث قبل الهجرة
```json
"date": {
  "hijri": 53,  // خطأ: يجب أن يكون -53
  "gregorian": 571
}
```

**صحيح**:
```json
"date": {
  "hijri": -53,
  "gregorian": 571
}
```

### أخطاء في الربط بين الأحداث والشخصيات

**خطأ**: استخدام اسم الصحابي بدلاً من id
```json
"companion_roles": [
  {
    "companion_id": "أبو بكر الصديق",  // خطأ: يجب استخدام id
    "role": "دوره"
  }
]
```

**صحيح**:
```json
"companion_roles": [
  {
    "companion_id": "abu-bakr",  // صحيح: استخدام id
    "role": "دوره"
  }
]
```

**خطأ**: استخدام id غير موجود
```json
"companion_id": "abu-bakr-siddiq"  // خطأ: id غير موجود
```

**صحيح**:
```json
"companion_id": "abu-bakr"  // صحيح: id موجود في companionsList.ts
```

### أخطاء في الـ aliases

```json
"aliases": ["عمر بن الخطاب"]  // خطأ: ناقص
```

```json
"aliases": [
  "عمر بن الخطاب",
  "عمر",
  "الفاروق",
  "أبو حفص"

```

### أخطاء في الـ id

**خطأ**: استخدام أحرف عربية أو مسافات
```json
"id": "غزوة بدر"  // خطأ: أحرف عربية ومسافات
```

**صحيح**:
```json
"id": "battle-of-badr"  // صحيح: إنجليزي بشرطات
```

**خطأ**: استخدام id مكرر
```json
// في حدث أول
"id": "badr"

// في حدث ثاني
"id": "badr"  // خطأ: مكرر
```

**صحيح**:
```json
// في حدث أول
"id": "battle-of-badr"

// في حدث ثاني
"id": "badr-aftermath"  // صحيح: فريد
```

## ملاحظات ختامية

1. **اختبر دائماً**: بعد إضافة أي محتوى، اختبره في المتصفح
2. **راجع المصادر**: تأكد من صحة المعلومات من مصادر موثوقة
3. **كن دقيقاً**: الدقة في التفاصيل مهمة جداً
4. **استخدم Git**: احفظ تغييراتك بانتظام
5. **اطلب المراجعة**: إذا كنت غير متأكد، اطلب من شخص آخر مراجعة عملك

للمزيد من المساعدة أو الاستفسارات، راجع الملفات الموجودة كأمثلة أو اطلب المساعدة من فريق التطوير.
